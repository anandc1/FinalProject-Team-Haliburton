Overview

The Online Restaurant Ordering System API is designed to help end-users interact with a comprehensive restaurant management system. This API provides functionality for managing restaurant operations including menu management, order processing, customer reviews, promotions, and business analytics.

Purpose and Primary Features:

  • Place and track customer orders with real-time status updates
  • Browse menu items organized by categories
  • Manage restaurant inventory and resources
  • Handle customer reviews and feedback
  • Apply promotional codes and discounts
  • Access business analytics and reporting

Setup Instructions

Step 1: Install Dependencies

Navigate to the project directory and install the required packages:

pip install fastapi
pip install "uvicorn[standard]"
pip install sqlalchemy
pip install pymysql
pip install pytest
pip install pytest-mock
pip install httpx
pip install cryptography

Step 2: Database Setup

1. Create a MySQL Database:

CREATE DATABASE sandwich_maker_api;

2. Configure Database Connection:
The API uses these default settings:

  • Host: localhost
  • Port: 3306
  • Username: root
  • Password: rootroot
  • Database: sandwich_maker_api

Step 3: Start the Server

uvicorn api.main:app --reload

Step 4: Verify Installation

Navigate to http://127.0.0.1:8000/docs to access the interactive API documentation.

Endpoint Documentation

Orders Module

POST /orders/guest

  • Purpose: Allow customers to place orders without account creation
  • Parameters: customer_name, customer_phone, customer_address, is_delivery, payment_method, items
  • Sample Request:
{
  "customer_name": "John Doe",
  "customer_phone": "555-0123",
  "customer_address": "123 Main St",
  "is_delivery": true,
  "payment_method": "card",
  "items": [{"dish_id": 1, "qty": 2}]
}

Expected Response: Order confirmation with tracking number and details

GET /orders/number/{order_number}

  • Purpose: Retrieve order details using tracking number
  • Parameters: order_number (path parameter)
  • Sample Request: GET /orders/number/ORD-20241201-001
  • Expected Response: Complete order information including items and status

PATCH /orders/{order_number}/status

  • Purpose: Update order status
  • Parameters: order_number (path), status (body)
  • Sample Request: {"status": "preparing"}
  • Expected Response: Updated order information

Menu Module

GET /menu/dishes

  • Purpose: Retrieve available menu items
  • Parameters: skip (optional), limit (optional), category_id (optional)
  • Sample Request: GET /menu/dishes?limit=10&category_id=1
  • Expected Response: List of dishes with details

POST /menu/dishes

  • Purpose: Add new menu item
  • Parameters: name, description, price_cents, category_id, available
  • Sample Request:
{
  "name": "Classic Burger",
  "description": "Beef patty with lettuce, tomato, and cheese",
  "price_cents": 1200,
  "category_id": 1,
  "available": true
}

Expected Response: Created dish information

Promotions Module

POST /promotions/apply

  • Purpose: Apply promotion code to calculate discount
  • Parameters: promotion_code, order_total_cents
  • Sample Request:
{
  "promotion_code": "SAVE10",
  "order_total_cents": 2500
}

Expected Response: Discount calculation with final amount

Reviews Module

POST /reviews/

  • Purpose: Submit customer feedback
  • Parameters: customer_name, rating, comment, order_id
  • Sample Request:
{
  "customer_name": "John Doe",
  "rating": 5,
  "comment": "Excellent food!",
  "order_id": 1
}

Expected Response: Created review information

Code Examples

Database Model Definition:

# api/models/orders.py
from sqlalchemy import Column, Integer, String, Boolean, Enum
from sqlalchemy.orm import relationship
import enum
from ..dependencies.database import Base

class OrderStatus(enum.Enum):
    PENDING = "pending"
    CONFIRMED = "confirmed"
    PREPARING = "preparing"
    READY = "ready"
    OUT_FOR_DELIVERY = "out_for_delivery"
    DELIVERED = "delivered"
    CANCELLED = "cancelled"

class Order(Base):
    __tablename__ = "orders"
    
    id = Column(Integer, primary_key=True, index=True, autoincrement=True)
    order_number = Column(String(50), unique=True, nullable=False)
    customer_name = Column(String(100), nullable=False)
    customer_phone = Column(String(20), nullable=False)
    status = Column(Enum(OrderStatus), default=OrderStatus.PENDING, nullable=False)
    total_cents = Column(Integer, nullable=False, default=0)
    
    # Relationships
    order_details = relationship("OrderDetail", back_populates="order")

FastAPI Router Implementation:

# api/routers/orders.py
from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.orm import Session
from ..controllers import orders as controller
from ..schemas import orders as schema
from ..dependencies.database import get_db

router = APIRouter(tags=['Orders'], prefix="/orders")

@router.post("/guest", response_model=schema.OrderOut)
def create_guest_order(request: schema.GuestOrderCreate, db: Session = Depends(get_db)):
    return controller.create_guest_order(db=db, payload=request)

@router.get("/number/{order_number}", response_model=schema.OrderOut)
def get_order_by_number(order_number: str, db: Session = Depends(get_db)):
    order = controller.get_order_by_number(db=db, order_number=order_number)
    if not order:
        raise HTTPException(status_code=404, detail="Order not found")
    return order

Setup Instructions:

  1. Clone repository and navigate to project directory
  2. Create virtual environment: python -m venv venv
  3. Activate virtual environment: source venv/bin/activate (Linux/Mac) or venv\Scripts\activate (Windows)
  4. Install dependencies from requirements.txt
  5. Configure database connection in api/dependencies/config.py
  6. Create MySQL database: CREATE DATABASE sandwich_maker_api;
  7. Run application: uvicorn api.main:app --reload
  8. Access documentation at: http://localhost:8000/docs
